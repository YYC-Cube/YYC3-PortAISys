### 2.2 任务类型配置

#### 2.2.1 创建任务类型

**时间**: 2026-06-02 上午  
**负责人**: 项目经理  
**参与人员**: 运维工程师×1  

**配置步骤**:
1. 进入项目设置 → "问题类型"
2. 创建以下任务类型：
   - Epic（史诗）
   - Story（用户故事）
   - Task（任务）
   - Bug（缺陷）
   - Sub-task（子任务）
3. 设置任务类型图标和描述
4. 设置任务类型字段
5. 点击"保存"

**任务类型列表**:
1. Epic（史诗）
   - 图标：📋
   - 描述：大型功能或项目里程碑
   - 字段：名称、描述、负责人、优先级、状态、开始时间、结束时间

2. Story（用户故事）
   - 图标：📝
   - 描述：用户需求或功能需求
   - 字段：名称、描述、负责人、优先级、状态、Epic、用户故事、验收标准

3. Task（任务）
   - 图标：✅
   - 描述：具体任务或工作项
   - 字段：名称、描述、负责人、优先级、状态、Story、开始时间、结束时间、交付物

4. Bug（缺陷）
   - 图标：🐛
   - 描述：软件缺陷或问题
   - 字段：名称、描述、负责人、优先级、状态、严重程度、复现步骤、预期结果、实际结果

5. Sub-task（子任务）
   - 图标：📌
   - 描述：任务的子任务
   - 字段：名称、描述、负责人、优先级、状态、父任务、开始时间、结束时间、交付物

**交付物**:

### 2.3 工作流配置

#### 2.3.1 创建工作流

**时间**: 2026-06-02 下午  
**负责人**: 项目经理  
**参与人员**: 运维工程师×1  

**配置步骤**:
1. 进入项目设置 → "工作流"
2. 创建工作流："P0-02-P0-03工作流"
3. 创建工作流状态：
   - 待办（To Do）
   - 进行中（In Progress）
   - 代码审查（Code Review）
   - 测试中（Testing）
   - 已完成（Done）
   - 已关闭（Closed）
4. 创建工作流转换：
   - 待办 → 进行中
   - 进行中 → 代码审查
   - 代码审查 → 进行中
   - 代码审查 → 测试中
   - 测试中 → 进行中
   - 测试中 → 已完成
   - 已完成 → 已关闭
   - 已完成 → 进行中
5. 设置转换条件和验证规则
6. 点击"发布"

**工作流状态**:
1. 待办（To Do）
   - 描述：任务已创建，等待开始
   - 图标：⏳

2. 进行中（In Progress）
   - 描述：任务正在进行中
   - 图标：🔄

3. 代码审查（Code Review）
   - 描述：代码已完成，等待审查
   - 图标：👀

4. 测试中（Testing）
   - 描述：代码已审查通过，正在测试
   - 图标：🧪

5. 已完成（Done）
   - 描述：任务已完成，等待验收
   - 图标：✅

6. 已关闭（Closed）
   - 描述：任务已验收通过，已关闭
   - 图标：🎉

**工作流转换**:
1. 待办 → 进行中
   - 转换名称：开始任务
   - 转换条件：任务负责人已分配
   - 验证规则：任务负责人必填

2. 进行中 → 代码审查
   - 转换名称：提交代码审查
   - 转换条件：代码已提交
   - 验证规则：代码提交链接必填

3. 代码审查 → 进行中
   - 转换名称：代码审查未通过
   - 转换条件：代码审查未通过
   - 验证规则：代码审查意见必填

4. 代码审查 → 测试中
   - 转换名称：代码审查通过
   - 转换条件：代码审查通过
   - 验证规则：代码审查通过意见必填

5. 测试中 → 进行中
   - 转换名称：测试未通过
   - 转换条件：测试未通过
   - 验证规则：测试失败原因必填

6. 测试中 → 已完成
   - 转换名称：测试通过
   - 转换条件：测试通过
   - 验证规则：测试报告必填

7. 已完成 → 已关闭
   - 转换名称：验收通过
   - 转换条件：验收通过
   - 验证规则：验收意见必填

8. 已完成 → 进行中
   - 转换名称：验收未通过
   - 转换条件：验收未通过
   - 验证规则：验收未通过原因必填

**交付物**:
1. 工作流创建完成
2. 工作流配置文档
