# YYC³ 综合测试框架

YYC³ 系统采用全面的测试策略，包括单元测试、集成测试、安全测试和性能测试，确保系统的质量、安全性和性能。

## 🧪 测试类型

### 1. 单元测试 (Unit Tests)

**位置**: `tests/unit/`

**目标**: 测试单个组件、函数和类的功能

**覆盖率目标**: ≥80%

**运行命令**:

```bash
pnpm test:unit
```

**包含的测试**:

- 核心组件测试 (AutonomousAIEngine, MessageBus, StateManager等)
- 智能体系统测试 (AgentManager, BaseAgent等)
- UI组件测试 (UIManager, LearningSystem等)
- 工具和辅助函数测试

### 2. 集成测试 (Integration Tests)

**位置**: `tests/integration/`

**目标**: 测试多个组件之间的交互和协同工作

**覆盖率目标**: ≥70%

**运行命令**:

```bash
pnpm test:integration
```

**包含的测试**:

- 系统集成测试 (SystemIntegration.test.ts)
- 智能体系统集成 (AgentSystemIntegration.integration.test.ts)
- 事件处理和错误处理集成 (EventDispatcherErrorHandler.integration.test.ts)
- 学习系统和内存集成 (LearningSystemMemory.integration.test.ts)

### 3. 安全测试 (Security Tests)

**位置**: `tests/security/`

**目标**: 验证系统的安全功能和防护机制

**运行命令**:

```bash
pnpm test:security
```

**包含的测试**:

- 认证和授权测试
- 输入验证和清理测试 (SQL注入、XSS、命令注入等)
- 威胁检测测试
- 合规性测试 (GDPR, SOC2等)
- 加密和数据保护测试

### 4. 性能测试 (Performance Tests)

**位置**: `tests/performance/`

**目标**: 验证系统性能指标是否达标

**运行命令**:

```bash
pnpm test:performance
```

**包含的测试**:

- 响应时间测试 (API响应 <200ms P95)
- 吞吐量测试 (QPS ≥1000)
- 资源使用测试 (CPU <70%, 内存 <80%)
- 并发处理测试
- 缓存性能测试
- 内存泄漏测试

## 🚀 快速开始

### 安装依赖

```bash
pnpm install
```

### 运行所有测试

```bash
# 运行所有测试并生成报告
pnpm test:all

# 或使用shell脚本
chmod +x scripts/run-all-tests.sh
./scripts/run-all-tests.sh
```

### 运行特定类型的测试

```bash
# 只运行单元测试
pnpm test:unit

# 只运行集成测试
pnpm test:integration

# 只运行安全测试
pnpm test:security

# 只运行性能测试
pnpm test:performance
```

### 运行测试覆盖率

```bash
pnpm test:coverage
```

### 查看测试UI

```bash
pnpm test:ui
```

## 📊 测试报告

测试完成后，会在 `test-reports/` 目录生成以下报告：

### 报告格式

1. **JSON报告** (`test-report.json`)
   - 机器可读的完整测试数据
   - 适合程序化处理和CI/CD集成

2. **Markdown报告** (`test-report.md`)
   - 人类可读的文本报告
   - 适合文档和代码仓库

3. **HTML报告** (`test-report.html`)
   - 交互式可视化报告
   - 适合在浏览器中查看

4. **覆盖率报告** (`coverage/index.html`)
   - 详细的代码覆盖率报告
   - 显示哪些代码被测试覆盖

### 查看报告

```bash
# 查看综合测试报告
open test-reports/test-report.html

# 查看覆盖率报告
open coverage/index.html
```

## 📈 性能指标

系统应该达到以下性能指标：

### 响应时间

- API响应时间: ≤200ms (P95)
- 页面加载时间: ≤2s (P95)
- 同步操作时间: ≤5s (P95)

### 吞吐量

- API QPS: ≥1000
- 并发用户数: ≥100
- 同步任务数: ≥50/min

### 可用性

- 系统可用性: ≥99.9%
- 数据一致性: ≥99.99%
- 错误率: ≤0.1%

### 资源使用

- CPU使用率: ≤70%
- 内存使用率: ≤80%
- 磁盘I/O: ≤80%

## 🛠️ CI/CD 集成

在CI/CD流水线中集成测试：

### GitHub Actions 示例

```yaml
name: Tests

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v3
      
      - uses: pnpm/action-setup@v2
        with:
          version: 8
      
      - uses: actions/setup-node@v3
        with:
          node-version: '18'
          cache: 'pnpm'
      
      - name: Install dependencies
        run: pnpm install
      
      - name: Run tests
        run: pnpm test:all
      
      - name: Upload coverage
        uses: codecov/codecov-action@v3
        with:
          files: ./coverage/coverage-final.json
      
      - name: Upload test reports
        uses: actions/upload-artifact@v3
        with:
          name: test-reports
          path: test-reports/
```

## 📝 编写测试

### 测试文件命名规范

- 单元测试: `*.test.ts`
- 集成测试: `*.integration.test.ts`
- 端到端测试: `*.e2e.test.ts`

### 测试结构

```typescript
import { describe, it, expect, beforeEach, afterEach } from 'vitest';

describe('组件名称', () => {
  // 设置
  beforeEach(() => {
    // 测试前的准备工作
  });

  // 清理
  afterEach(() => {
    // 测试后的清理工作
  });

  describe('功能描述', () => {
    it('应该能够做某事', () => {
      // 测试代码
      expect(result).toBe(expected);
    });
  });
});
```

### 测试最佳实践

1. **测试独立性**: 每个测试应该独立运行，不依赖其他测试
2. **清晰的描述**: 测试名称应该清楚描述测试的内容
3. **单一职责**: 每个测试只测试一个功能点
4. **使用Mock**: 合理使用模拟对象避免外部依赖
5. **覆盖边界**: 测试边界条件和异常情况
6. **性能考虑**: 测试应该快速执行
