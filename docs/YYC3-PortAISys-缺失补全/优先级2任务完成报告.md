**核心权限**:
```typescript
export enum Permission {
  MAPPING_READ = 'mapping:read',
  MAPPING_WRITE = 'mapping:write',
  MAPPING_DELETE = 'mapping:delete',
  SYNC_READ = 'sync:read',
  SYNC_EXECUTE = 'sync:execute',
  SYNC_MANAGE = 'sync:manage',
  ALERT_READ = 'alert:read',
  ALERT_MANAGE = 'alert:manage',
  USER_READ = 'user:read',
  USER_WRITE = 'user:write',
  USER_DELETE = 'user:delete',
  SETTINGS_READ = 'settings:read',
  SETTINGS_WRITE = 'settings:write',
  ADMIN = 'admin',
}
```

**角色定义**:
- ADMIN（管理员）：拥有所有权限
- MODERATOR（版主）：拥有映射、同步、告警、用户、设置的部分权限
- USER（用户）：拥有查看映射、同步、告警的权限
- GUEST（访客）：仅拥有查看映射的权限

## 📊 技术实现细节

### 权限管理系统架构

```
src/types/permissions.ts
    ↓
src/lib/permissions.ts (PermissionManager)
    ↓
src/hooks/use-permissions.ts (usePermissions)
    ↓
src/components/permission-guard.tsx (PermissionGuard)
    ↓
src/app/users/page.tsx (用户管理)
src/app/mappings/page.tsx (映射管理)
```

### 权限检查流程

1. **用户登录** → NextAuth.js创建Session
2. **Session Provider** → 提供用户信息到整个应用
3. **usePermissions Hook** → 从Session获取用户角色
4. **PermissionManager** → 根据角色获取权限列表
5. **PermissionGuard** → 检查用户是否拥有所需权限
6. **UI渲染** → 根据权限显示/隐藏功能
