## 三、部署步骤

### 3.1 准备工作

#### 3.1.1 环境检查

**检查脚本**:
```bash
#!/bin/bash

echo "🔍 环境检查"
echo "=============="

# 检查操作系统
OS=$(uname -s)
echo "操作系统: $OS"

# 检查CPU
CPU_CORES=$(nproc 2>/dev/null || sysctl -n hw.ncpu 2>/dev/null || echo "Unknown")
echo "CPU核心数: $CPU_CORES"

# 检查内存
MEMORY=$(free -h 2>/dev/null | awk '/^Mem:/ {print $2}' || echo "Unknown")
echo "内存: $MEMORY"

# 检查磁盘空间
DISK=$(df -h / 2>/dev/null | awk 'NR==2 {print $4}' || echo "Unknown")
echo "可用磁盘空间: $DISK"

# 检查Node.js
if command -v node &> /dev/null; then
    NODE_VERSION=$(node -v)
    echo "Node.js版本: $NODE_VERSION"
    
    # 检查版本是否满足要求
    NODE_MAJOR=$(echo $NODE_VERSION | cut -d'v' -f2 | cut -d'.' -f1)
    if [ "$NODE_MAJOR" -lt 18 ]; then
        echo "❌ Node.js版本过低，需要18.0.0或更高"
        exit 1
    else
        echo "✅ Node.js版本满足要求"
    fi
else
    echo "❌ 未找到Node.js"
    exit 1
fi

# 检查npm
if command -v npm &> /dev/null; then
    NPM_VERSION=$(npm -v)
    echo "npm版本: $NPM_VERSION"
else
    echo "❌ 未找到npm"
    exit 1
fi

echo "✅ 环境检查完成"
```

#### 3.1.2 依赖安装

**安装脚本**:
```bash
#!/bin/bash

echo "📦 依赖安装"
echo "============"

# 更新包管理器
if command -v apt-get &> /dev/null; then
    echo "更新apt包管理器..."
    sudo apt-get update
elif command -v yum &> /dev/null; then
    echo "更新yum包管理器..."
    sudo yum update -y
elif command -v brew &> /dev/null; then
    echo "更新Homebrew..."
    brew update
fi

# 安装Node.js（如果未安装）
if ! command -v node &> /dev/null; then
    echo "安装Node.js..."
    if command -v apt-get &> /dev/null; then
        sudo apt-get install -y curl
        curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
        sudo apt-get install -y nodejs
    elif command -v yum &> /dev/null; then
        sudo yum install -y curl
        curl -fsSL https://rpm.nodesource.com/setup_18.x | sudo bash -
        sudo yum install -y nodejs npm
    elif command -v brew &> /dev/null; then
        brew install node
    fi
fi

# 安装Git（如果未安装）
if ! command -v git &> /dev/null; then
    echo "安装Git..."
    if command -v apt-get &> /dev/null; then
        sudo apt-get install -y git
    elif command -v yum &> /dev/null; then
        sudo yum install -y git
    elif command -v brew &> /dev/null; then
        brew install git
    fi
fi

echo "✅ 依赖安装完成"
```

### 3.2 系统部署

#### 3.2.1 获取源码

**部署脚本**:
```bash
#!/bin/bash

echo "📥 获取源码"
echo "============"

# 设置部署目录
DEPLOY_DIR="/opt/yyc3-doc-sync"
SERVICE_USER="yyc3"

# 创建服务用户
if ! id "$SERVICE_USER" &>/dev/null; then
    echo "创建服务用户: $SERVICE_USER"
    sudo useradd -r -s /bin/false $SERVICE_USER
fi

# 创建部署目录
sudo mkdir -p $DEPLOY_DIR
sudo chown $SERVICE_USER:$SERVICE_USER $DEPLOY_DIR

# 获取源码
if [ -d ".git" ]; then
    echo "从Git仓库拉取最新代码..."
    sudo -u $SERVICE_USER git clone https://github.com/yyc3/doc-code-sync.git $DEPLOY_DIR
else
    echo "复制本地代码..."
    sudo cp -r . $DEPLOY_DIR
    sudo chown -R $SERVICE_USER:$SERVICE_USER $DEPLOY_DIR
fi

cd $DEPLOY_DIR
echo "✅ 源码获取完成"
```

#### 3.3.2 服务配置

**配置脚本**:
```bash
#!/bin/bash

echo "⚙️ 服务配置"
echo "=============="

cd $DEPLOY_DIR

# 创建systemd服务文件
sudo tee /etc/systemd/system/yyc3-doc-sync.service > /dev/null << 'EOF'
[Unit]
Description=YYC³ 文档同步工具
After=network.target

[Service]
Type=simple
User=yyc3
WorkingDirectory=/opt/yyc3-doc-sync
ExecStart=/usr/bin/node /opt/yyc3-doc-sync/dist/index.js watch
Restart=always
RestartSec=10
Environment=NODE_ENV=production

[Install]
WantedBy=multi-user.target
EOF

# 重新加载systemd配置
sudo systemctl daemon-reload

# 启用服务
sudo systemctl enable yyc3-doc-sync.service

echo "✅ 服务配置完成"
```

**监控脚本**:
```bash
#!/bin/bash

echo "📊 监控检查"
echo "============"

# 检查服务状态
SERVICE_STATUS=$(sudo systemctl is-active yyc3-doc-sync.service)
echo "服务状态: $SERVICE_STATUS"
```

#### 4.2.2 告警处理

**告警脚本**:
```bash
#!/bin/bash

echo "🚨 告警处理"
echo "============"

# 检查服务状态
if ! sudo systemctl is-active --quiet yyc3-doc-sync.service; then
    echo "🚨 服务告警: 主服务未运行"

    curl -X POST -H 'Content-type: application/json' \
        --data '{"text":"🚨 YYC³文档同步工具告警: 主服务未运行"}' \

# 检查CPU使用率

if (( $(echo "$CPU_USAGE > 85" | bc -l) )); then
    echo "🚨 资源告警: CPU使用率过高 (${CPU_USAGE}%)"
    # 发送告警通知
    curl -X POST -H 'Content-type: application/json' \
        --data "{\"text\":\"🚨 YYC³文档同步工具告警: CPU使用率过高 (${CPU_USAGE}%)\"}" \
        $ALERT_WEBHOOK_URL
fi

# 检查内存使用率
MEMORY_USAGE=$(free | grep Mem | awk '{printf("%.1f", $3/$2 * 100.0)}')
if (( $(echo "$MEMORY_USAGE > 90" | bc -l) )); then
    echo "🚨 资源告警: 内存使用率过高 (${MEMORY_USAGE}%)"
    # 发送告警通知
    curl -X POST -H 'Content-type: application/json' \
        --data "{\"text\":\"🚨 YYC³文档同步工具告警: 内存使用率过高 (${MEMORY_USAGE}%)\"}" \
        $ALERT_WEBHOOK_URL
fi

# 检查磁盘使用率
DISK_USAGE=$(df / | awk 'NR==2 {print $5}' | cut -d'%' -f1)
if [ "$DISK_USAGE" -gt 90 ]; then
    echo "🚨 资源告警: 磁盘使用率过高 (${DISK_USAGE}%)"

fi

echo "✅ 告警检查完成"
```

```bash
    echo "恢复日志文件..."

    sudo tar -xzf $BACKUP_DIR/logs_$BACKUP_DATE.tar.gz -C /opt/yyc3-doc-sync/
```

## 五、故障排除

### 5.1 常见问题

#### 5.1.1 服务启动失败

**问题**: 服务启动失败

**排查步骤**:
1. 检查服务状态
   ```bash
   sudo systemctl status yyc3-doc-sync.service
   ```

2. 查看服务日志
   ```bash
   sudo journalctl -u yyc3-doc-sync.service -n 50
   ```

3. 检查端口占用
   ```bash
   sudo netstat -tuln | grep 3100
   ```

4. 检查文件权限
   ```bash
   ls -la /opt/yyc3-doc-sync/
   ```

**解决方案**:
- 修复配置错误
- 释放占用端口
- 调整文件权限
- 重启服务

#### 5.1.2 性能问题

**问题**: 系统性能下降

**排查步骤**:
1. 检查系统资源
   ```bash
   top
   free -h
   df -h
   ```

2. 检查进程状态
   ```bash
   ps aux | grep node
   ```

3. 检查网络连接
   ```bash
   netstat -an | grep ESTABLISHED | wc -l
   ```

**解决方案**:
- 优化配置参数
- 增加系统资源
- 清理日志文件
- 重启服务

#### 5.1.3 同步失败

**问题**: 文档同步失败

**排查步骤**:
1. 检查映射配置
   ```bash
   cat /opt/yyc3-doc-sync/.doc-code-mapping.json
   ```

2. 检查文件权限
   ```bash
   ls -la /path/to/docs
   ls -la /path/to/code
   ```

3. 查看同步日志
   ```bash
   sudo -u yyc3 tail -f /opt/yyc3-doc-sync/logs/sync.log
   ```

**解决方案**:
- 修复映射规则
- 调整文件权限
- 检查磁盘空间
- 重启同步服务

### 5.2 应急处理

#### 5.2.1 服务崩溃

**处理流程**:
1. 立即重启服务
   ```bash
   sudo systemctl restart yyc3-doc-sync.service
   ```

2. 检查崩溃原因
   ```bash
   sudo journalctl -u yyc3-doc-sync.service -n 100
   ```

3. 保存现场数据
