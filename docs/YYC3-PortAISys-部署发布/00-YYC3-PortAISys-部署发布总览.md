## 🎯 部署发布概述

YYC³ PortAISys 采用现代化的部署发布策略，支持多种部署方式，从本地开发到生产环境的完整部署生命周期管理。

### 部署发布目标

- **可靠性**: 确保部署过程稳定可靠，零停机部署
- **自动化**: 最大化自动化程度，减少人工干预
- **可追溯**: 完整的部署历史和版本追溯
- **快速回滚**: 快速回滚能力，最小化故障影响
- **环境一致性**: 确保各环境配置一致性

## 🚀 快速开始

### 前置要求

| 要求 | 版本 | 说明 |
|------|------|------|
| **Node.js** | >= 20.19.0 | 运行时环境 |
| **npm/pnpm** | >= 9.0.0 | 包管理器 |
| **PostgreSQL** | >= 15.0 | 数据库 |
| **Redis** | >= 7.0 | 缓存 |
| **Docker** | Latest | 容器化（可选） |

### 快速部署

```bash
# 1. 克隆项目
git clone https://github.com/YYC-Cube/YYC3-PortAISys.git
cd YYC3-PortAISys

# 2. 安装依赖
pnpm install

# 3. 配置环境
cp .env.example .env
# 编辑 .env 文件

# 4. 初始化数据库
npx prisma migrate deploy
npx prisma db seed

# 5. 启动服务
pnpm dev
```

## 🚨 故障处理

### 常见部署问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| **构建失败** | 依赖冲突/类型错误 | 检查依赖、修复类型错误 |
| **启动失败** | 配置错误/端口冲突 | 检查配置、释放端口 |
| **数据库连接失败** | 连接串错误/网络问题 | 检查连接串、网络 |
| **内存不足** | 资源限制 | 增加内存配额 |

### 紧急回滚

```bash
# 快速回滚到上一版本
kubectl rollout undo deployment/yyc3-api

# 回滚到指定版本
kubectl rollout undo deployment/yyc3-api --to-revision=3

# 查看回滚状态
kubectl rollout status deployment/yyc3-api
```
