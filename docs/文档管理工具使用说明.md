# 文档管理工具使用说明


## 📋 概述

YYC³ Portable Intelligent AI System 文档管理工具集提供自动化的文档同步和合规化功能，确保文档体系始终保持最新状态和符合团队标准。

---

## 🛠️ 工具列表

### 1. 文档自动同步工具

**脚本**: `scripts/doc-sync-tool.ts`

**功能**:
- 自动扫描docs目录下的所有文档
- 生成各目录的映射总结文档
- 更新根目录的文档索引总览
- 保持文档结构的同步和一致性

**使用方法**:

```bash
# 执行文档同步
npm run docs:sync

# 或使用tsx直接运行
tsx scripts/doc-sync-tool.ts
```

**输出结果**:
- 更新 `docs/文档索引总览.md`
- 更新各子目录的 `目录映射总结.md`
- 显示同步统计信息

---

### 2. 文档标头标尾合规化工具

**脚本**: `scripts/doc-compliance-tool.ts`

**功能**:
- 自动检查文档标头标尾合规性
- 添加或更新符合YYC³标准的标头
- 添加或更新符合YYC³标准的标尾
- 生成合规性统计报告

**使用方法**:

```bash
# 检查文档合规性（不修改文件）
npm run docs:compliance:check

# 修复文档合规性问题
npm run docs:compliance:fix

# 或直接使用
npm run docs:compliance
```

**命令行参数**:

| 参数 | 说明 |
|------|------|
| `--dry-run` | 只检查不修改文件 |
| `--verbose` | 显示详细信息 |

## 🎯 最佳实践

### 文档命名规范

- **格式**: `序号-YYC3-PortAIsys-文档名称.md`
- **序号**: 两位数字，按顺序排列
- **名称**: 清晰描述文档内容
- **扩展名**: `.md` (Markdown格式)

**示例**:
- `01-YYC3-PortAIsys-快速开始指南.md`
- `02-YYC3-PortAIsys-安装部署指南.md`
- `03-YYC3-PortAIsys-配置管理指南.md`

### 文档结构规范

每个文档应包含：

1. **文档标题**: 清晰的标题
2. **文档描述**: 简要描述
3. **文档内容**: 详细内容
4. **相关链接**: 相关文档链接
5. **更新记录**: 版本更新记录

### 定期维护

建议定期执行以下操作：

- **每日**: 添加新文档后执行 `npm run docs:sync`
- **每周**: 执行 `npm run docs:compliance:check` 检查合规性
- **每月**: 执行 `npm run docs:compliance:fix` 确保所有文档符合标准

## 🔧 故障排除

### 常见问题

**Q: 文档同步后索引没有更新？**

A: 确保在docs目录下有以`YYC3-PortAIsys-`开头的子目录，并且目录中包含Markdown文件。

**Q: 合规化工具修改了不需要修改的文档？**

A: 使用 `--dry-run` 参数先检查，确认无误后再执行修复操作。

**Q: 标头标尾格式不符合预期？**

A: 检查文档内容是否已经有标头标尾，工具会跳过已有有效标头标尾的文档。
